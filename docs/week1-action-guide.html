本周目标

到第 7 天结束，你应该能主动掌控一场短链接系统设计面试：先定范围，再给 API 和核心实体，再用容量估算判断是否真的需要分片，最后把快速跳转和唯一短码生成两个深挖点讲到可辩护。

算法训练

每天三题位

第 1 周只刷 NeetCode 150 的 Arrays & Hashing：核心题、同类扩展、限时复盘依次推进。下一组 tag 留到后续周。

知识边界

这一周的范围必须收紧。你不是在泛读 Hello Interview、DDIA、Grokking、Alex Xu，而是在把它们中最该进入 Bitly 答案的部分组合成可交付能力。

必须掌握
  • Hello Interview: Delivery Framework 的 45 分钟结构。
  • 功能需求与量化非功能需求的写法。
  • REST 资源、HTTP 方法、状态码、幂等性、从认证上下文取得身份。
  • 从访问路径推数据模型：URL 映射、唯一索引、过期时间、缓存 TTL。
  • 容量判断：10 亿条 URL 映射约 500GB，本身不足以推出分片。
  • 跳转路径：DNS / 负载均衡或 CDN / 读服务 / 缓存 / 数据库 / 302。
  • 算法按 NeetCode tag 吸收：先掌握分类内核心题，再扩展同类热门题，不做随机刷题。
本周刻意不展开
  • 完整点击分析链路：地域、来源、设备、滥用检测。
  • 完整多地域双活架构，只作为高级扩展点提一句。
  • Kafka、Flink、事件溯源深挖，留到指标、广告、支付系统。
  • Grokking / Alex Xu 大范围平行阅读。本周只用精准源包。
  • 背图。每个盒子和箭头都必须能从 API 与状态变化推出来。

时间线

顺序是刻意安排的：先学会面试里的操作节奏，再把每个概念绑定到 Bitly，最后在时间压力下跑模拟面试。每一天都有直达链接和验收标准。
